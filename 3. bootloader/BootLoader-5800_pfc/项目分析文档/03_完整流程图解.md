## 系统启动流程详解

### 1. 启动序列时间轴

```
时间    事件                          状态
────────────────────────────────────────────────
T=0ms   硬件复位 (Reset Pin)
        └─ 处理器加载复位向量

T=1us   Reset_Handler()
        ├─ 初始化数据段
        ├─ 设置堆栈指针
        └─ 跳转到SystemInit()

T=2us   SystemInit() (CMSIS)
        ├─ 设置中断向量表 (VTOR)
        ├─ 配置FPU浮点单元
        └─ 返回到Reset_Handler

T=3us   __PROGRAM_START()
        ├─ 调用全局构造函数
        └─ 跳转到main()

T=5us   main()
        ├─ SystemClock_Config()
        │   ├─ 配置PLL0 (12MHz → 180MHz)
        │   ├─ 配置PLL1 (12MHz → 360MHz)
        │   ├─ 配置各外设时钟
        │   └─ 更新SystemCoreClock
        │
        ├─ LL_Init()
        │   └─ 底层库初始化
        │
        ├─ delayms(20)
        │   └─ 延时20ms等待系统稳定
        │       └─ [喂狗1次: iwdg_feed()]
        │
        └─ sys_init()
            ├─ iwdg_init(1000)
            │   ├─ 配置看门狗超时1000ms
            │   └─ 启动看门狗
            │
            ├─ led_init()
            │   └─ GPIO PC13初始化为输出
            │
            ├─ iap_init()
            │   └─ 清零IAP标志位
            │
            ├─ modbus_iap_init()
            │   ├─ modbuss_init(addr=2)
            │   ├─ modbuss_set_timeout(1000ms)
            │   ├─ modbuss_set_iap_callback()
            │   ├─ [UART模式]
            │   │   ├─ modbuss_set_send_callback(uart1_send)
            │   │   └─ uart1_init(115200)
            │   │       ├─ LL_UART_Init()
            │   │       └─ ring_buff_init()
            │   └─ [CAN模式]
            │       ├─ modbuss_set_send_callback(cansmb_send)
            │       └─ cansmb_init(id=0xAA55, rate=125k)
            │           ├─ cans_init()
            │           │   └─ LL_CAN_Init()
            │           └─ cans_set_recv_callback()
            │
            └─ timer8_init()
                ├─ 配置TMR8 20us周期
                ├─ LL_TMR_Init()
                ├─ LL_TMR_Base_Cfg()
                └─ LL_TMR_Start_IT()
                    └─ 启用TMR8中断

T=25ms  进入主循环 while(1)
        └─ 系统完全初始化完成
```

### 3. 时钟配置流程

```
外部晶振 12MHz
    ▼
┌────────────────┐
│  时钟配置模块   │
│SystemClock_    │
│Config()        │
└────────────────┘
    ├─► LL_RCU_SysclkInit()
    │    └─► PLL0 × 15 = 180MHz (SYSCLK)
    │
    ├─► SystemCoreClockUpdate()
    │    └─► SystemCoreClock = 180000000
    │
    ├─► LL_RCU_Pll1Cfg()
    │    └─► PLL1 × 30 = 360MHz
    │
    ├─► LL_RCU_HRPWM_ClkCfg()
    │    └─► PLL1 ÷ 2 = 180MHz (HRPWM)
    │
    ├─► LL_RCU_ADC_ClkCfg()
    │    └─► PLL1 ÷ 6 = 60MHz (ADC)
    │
    └─► LL_RCU_CAN_ClkCfg()
         └─► PLL1 ÷ 3 = 120MHz (CAN)

最终时钟树:
    ┌─────────────────────────────────┐
    │      外部晶振 12MHz              │
    └─────────────────────────────────┘
                    ▼
    ┌─────────────────────────────────┐
    │  PLL0 倍频×15 → 180MHz          │
    │  用于 SYSCLK (系统主时钟)        │
    └─────────────────────────────────┘
                    ▼
    ┌─────────────────────────────────┐
    │  系统主频: 180MHz               │
    │  用于 CPU 内核                  │
    └─────────────────────────────────┘

    ┌─────────────────────────────────┐
    │  PLL1 倍频×30 → 360MHz          │
    │  母频用于外设分频                │
    └─────────────────────────────────┘
         ┌──────┬──────┬──────┐
         ▼      ▼      ▼      ▼
      ÷2   ÷3   ÷6   其他
      180  120  60
      MHz  MHz  MHz
       │    │    │
       ▼    ▼    ▼
     HRPWM CAN  ADC
```

## 主循环详解

### 1. 主循环时序图

```
每个主循环周期 (~1-10ms):

┌─────────────────────────────────────────────────┐
│                  while(1)                        │
│              主循环开始                          │
└─────────────────────────────────────────────────┘
                    ▼
     ┌──────────────────────────────────┐
     │   Task 1: modbus_iap_task()      │
     │   (Modbus IAP通信任务)           │
     │   └─ [UART模式]                  │
     │      ├─ uart1_rx_poll()          │
     │      ├─ modbuss_recv_byte()      │
     │      └─ modbuss_task()           │
     └──────────────────────────────────┘
                    ▼
     ┌──────────────────────────────────┐
     │   Task 2: iap_task()             │
     │   (IAP处理任务)                  │
     │   ├─ 超时检查                    │
     │   ├─ Flash验证                   │
     │   └─ 跳转APP                     │
     └──────────────────────────────────┘
                    ▼
     ┌──────────────────────────────────┐
     │   Task 3: sys_task()             │
     │   (系统维护任务)                 │
     │   └─ iwdg_feed() [喂狗]          │
     └──────────────────────────────────┘
                    ▼
     ┌──────────────────────────────────┐
     │   Task 4: 通信轮询               │
     │   ├─ [UART模式]                  │
     │   │  └─ uart1_tx_poll()          │
     │   └─ [CAN模式]                   │
     │      └─ cans_rx_poll()           │
     └──────────────────────────────────┘
                    ▼
          返回主循环顶部
          (循环开始)

┌─────────────────────────────────┐
│  后台中断处理 (不阻断主循环)     │
│  ├─ TMR8中断 (每20us)           │
│  │  └─ led_task() (每100us)    │
│  └─ UART/CAN接收中断           │
└─────────────────────────────────┘
```

## IAP更新流程详解

### 1. 完整的IAP升级时序图

```
上位机                    BootLoader                 Flash
  │                           │                       │
  │ ─────── cmd=1 ─────────►  │                       │
  │  (进入IAP)                │                       │
  │                     cmd_enter_iap()              │
  │                           ├─ iap_flag=true       │
  │                           │                       │
  │ ◄─── ACK(cmd=1) ────────  │                       │
  │                                                   │
  │ ─────── cmd=3 ─────────►  │                       │
  │  (写Flash-数据块1)         │                       │
  │  [0x08008000, 256字节]    │                       │
  │                     cmd_write_flash()            │
  │                           ├─ flash_program()     │
  │                           │   ├─ 擦除扇区        │ 擦除
  │                           │   ├─ 写数据          │ 写数据
  │                           │   └─ 验证            │ 验证
  │                           ├─ size=0 (成功)      │
  │                                                   │
  │ ◄─── ACK(cmd=3,size=0)─  │                       │
  │                                                   │
  │ ─────── cmd=3 ─────────►  │  [重复256字节块]    │
  │  (写Flash-数据块2)         │  直到整个固件       │
  │  [0x08008100, 256字节]    │  传输完成           │
  │                                                   │
  │ ◄─── ACK(cmd=3,size=0)─  │                       │
  │                                                   │
  │  ...  (更多数据块)  ...                          │
  │                                                   │
  │ ─────── cmd=4 ─────────►  │                       │
  │  (写校验信息)              │                       │
  │  data[0:3]=大小           │                       │
  │  data[4:7]=CRC32          │                       │
  │                     cmd_write_checksum()         │
  │                           ├─ flash_program()     │
  │                           │ (写ARG_BASE_ADDR)    │ 写校验
  │                                                   │
  │ ◄─── ACK(cmd=4,size=0)─  │                       │
  │                                                   │
  │ ─────── cmd=0 ─────────►  │                       │
  │  (退出IAP)                 │                       │
  │                     cmd_exit_iap()               │
  │                           ├─ iap_flash_verify()  │
  │                           │   └─ crc32_update()  │
  │                           │   └─ 比较CRC32      │
  │                           ├─ jump_flag=true     │
  │                                                   │
  │ ◄─── ACK(cmd=0) ────────  │                       │
  │                                                   │
  │                     [下次循环]                    │
  │                     iap_task()                    │
  │                           ├─ 检测jump_flag     │
  │                           ├─ 禁用中断/外设      │
  │                           └─ jump_to_app()      │
  │                              (不返回)            │
  │                                                   │
  │                    应用程序开始执行              │
```

### 2. IAP命令决策树

```
接收到IAP数据包
        │
        ▼
   ┌─────────────┐
   │ 命令 = ?    │
   └─────────────┘
        │
    ┌───┼───┬───┬────┐
    │   │   │   │    │
   0   1   2   3    4
    │   │   │   │    │
    ▼   ▼   ▼   ▼    ▼
  Exit Enter Read Write Check
  IAP  IAP  Flash Flash Sum
    │   │    │    │    │
    │   │    │    │    └─ cmd_write_checksum()
    │   │    │    │       ├─ 写app_size
    │   │    │    │       ├─ 写app_crc32
    │   │    │    │       └─ 返回ACK
    │   │    │    │
    │   │    │    └─ cmd_write_flash()
    │   │    │       ├─ flash_program()
    │   │    │       ├─ 擦除/写入/验证
    │   │    │       └─ 返回ACK
    │   │    │
    │   │    └─ cmd_read_flash()
    │   │       ├─ memmove()
    │   │       ├─ 读取数据
    │   │       └─ 返回数据
    │   │
    │   └─ cmd_enter_iap()
    │      ├─ 设置iap_flag=true
    │      ├─ 停止自动超时
    │      └─ 返回ACK
    │
    └─ cmd_exit_iap()
       ├─ iap_flash_verify()
       │  ├─ 读app_size和crc32
       │  ├─ 计算crc32
       │  └─ 比较是否相等
       ├─ 成功?
       │  ├─ YES: 设置jump_flag
       │  │       返回ACK
       │  └─ NO: 返回失败(cmd=0xFFFF)
       └─ 下次循环跳转APP
          或继续等待IAP命令
```

## 通信流程详解

### 1. UART通信流程

```
UART接收中断(硬件)
    │
    ▼
┌─────────────────────────────────┐
│ UART1接收缓冲区有数据           │
│ (可能多个字节已接收)            │
└─────────────────────────────────┘
    │
    │ (在主循环中)
    ▼
┌─────────────────────────────────┐
│ modbus_iap_task()               │
│ ├─ while(uart1_rx_poll(&data))  │
│ │  ├─ 检查UART接收状态          │
│ │  ├─ 如果有数据:               │
│ │  │  └─ data = 读UART          │
│ │  └─ 返回true/false            │
│ └─ 循环直到无数据               │
└─────────────────────────────────┘
    │
    ▼ (逐字节处理)
┌─────────────────────────────────┐
│ modbuss_recv_byte(data)          │
│ ├─ 使用状态机处理               │
│ ├─ 累积字节到缓冲区             │
│ └─ 当完整包接收后:              │
│     └─ 设置pkt_flag=true        │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│ modbuss_task()                  │
│ ├─ 检查pkt_flag                 │
│ ├─ 如果true:                    │
│ │  ├─ pkt_decode()              │
│ │  │  ├─ CRC16校验              │
│ │  │  ├─ 地址匹配检查           │
│ │  │  └─ 根据fno调用回调        │
│ │  └─ 清除pkt_flag              │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│ on_iap_cmd() (回调)             │
│ ├─ 提取IAP命令                  │
│ ├─ iap_pkt_decode()             │
│ │  └─ 分发到cmd处理函数         │
│ └─ modbuss_send_iap()           │
│     └─ 发送响应                 │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│ uart1_send_data(data, size)     │
│ ├─ ring_buff_save_data()        │
│ │  └─ 数据加入发送缓冲          │
└─────────────────────────────────┘
    │
    │ (在主循环中)
    ▼
┌─────────────────────────────────┐
│ uart1_tx_poll()                 │
│ ├─ 检查TxFIFO是否满             │
│ ├─ ring_buff_take_byte()        │
│ │  └─ 从缓冲取一字节             │
│ └─ 写入UART发送寄存器           │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│ UART发送中断                    │
│ └─ 硬件逐位发送数据             │
└─────────────────────────────────┘
    │
    ▼
上位机接收数据 (完成通信)
```

### 2. CAN通信流程

```
CAN接收中断(硬件)
    │
    ▼
┌─────────────────────────────────┐
│ CAN邮箱接收到数据               │
│ (8字节/帧)                      │
└─────────────────────────────────┘
    │
    │ (在中断或轮询中)
    ▼
┌─────────────────────────────────┐
│ cans_rx_poll()                  │
│ ├─ LL_CAN_Receive_CPU()         │
│ │  └─ 读CAN邮箱数据             │
│ └─ on_recv() (回调)             │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│ on_can_recv(id, data, size)     │
│ ├─ 检查CAN ID (ID过滤)          │
│ ├─ 如果ID匹配:                  │
│ │  └─ for(i=0; i<size; i++)     │
│ │      └─ modbuss_recv_byte()   │
│ └─ [其他ID: 忽略]               │
└─────────────────────────────────┘
    │
    ▼ (Modbus处理流程同UART)
┌─────────────────────────────────┐
│ modbuss_task() → 处理完整包      │
│ → on_iap_cmd() → cmd处理        │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│ modbuss_send_iap()              │
│ ├─ 组装Modbus包                 │
│ └─ on_send() (发送回调)         │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│ cansmb_send_data(data, size)    │
│ ├─ 分包: size_per_frame = 8     │
│ ├─ for(每个包) {                │
│ │  ├─ 取8字节数据               │
│ │  ├─ cans_send_data()          │
│ │  │  └─ LL_CAN_TransmitPTB()   │
│ │  ├─ 等待发送完成              │
│ │  │  └─ while(cans_tx_is_busy) │
│ │  │     └─ iwdg_feed()         │
│ │  └─ 继续发送下一包            │
│ │ }                              │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│ CAN硬件发送中断                 │
│ └─ 逐位发送CAN帧                │
└─────────────────────────────────┘
    │
    ▼
上位机接收CAN帧 (完成通信)
```

## 中断处理详解

### 1. 定时器中断流程 (TMR8, 20us周期)

```
硬件时钟 (180MHz)
    │
    ▼
TMR8分频器
    └─► 计数到预设值 (20us周期)

    ▼
┌─────────────────────────────────┐
│ TMR8溢出中断 (50KHz)            │
│ 触发频率: 每20微秒一次          │
└─────────────────────────────────┘
    │
    ▼ (中断处理程序)
┌─────────────────────────────────┐
│ LL_TMR_OverflowCallback()        │
│ (由HAL库自动调用)               │
│ └─ 调用 sys_timer_isr()         │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│ sys_timer_isr()                 │
│ ├─ 递增计数器 (cnt++)           │
│ ├─ 每5次调用一次LED任务         │
│ │  └─ if (cnt % 5 == 0)         │
│ │      └─ led_task()            │
│ └─ 计数器周期 = 256             │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│ led_task() (每100us调用)         │
│ ├─ 递增led计数器                │
│ ├─ 每5000次调用翻转LED          │
│ │  └─ if (led_cnt >= 5000)      │
│ │      ├─ led_cnt = 0           │
│ │      └─ pc_toggle_pin(13)     │
│ └─ LED周期 = 100us * 5000 = 500ms
└─────────────────────────────────┘

中断周期汇总:
    ├─ TMR8中断:   20us   (50KHz)
    ├─ sys_timer_isr: 100us (10KHz)
    ├─ led_task:  100us  (每100us被调用1次)
    └─ LED切换:   500ms  (实际闪烁周期)
```

### 2. UART接收中断流程

```
UART1接收到数据
    │
    ▼
┌──────────────────────────┐
│ UART RxFIFO非空中断      │
└──────────────────────────┘
    │
    ▼ (中断处理程序)
┌──────────────────────────┐
│ UART1_RX_IRQ Handler     │
│ (HAL库或驱动程序提供)    │
│ ├─ 读UART接收数据寄存器  │
│ ├─ 数据存入接收FIFO      │
│ └─ 中断返回              │
└──────────────────────────┘
    │
    ▼ (在主循环中)
┌──────────────────────────┐
│ modbus_iap_task()        │
│ └─ uart1_rx_poll()       │
│    ├─ 检查FIFO           │
│    ├─ 读一个字节         │
│    └─ 返回该字节         │
└──────────────────────────┘
    │
    ▼ (处理该字节)
Modbus状态机处理...
```

### 3. 看门狗超时中断

```
硬件时钟
    │
    ▼
独立看门狗计时器
    └─► 计数到1000ms (预设超时)

    ▼
┌─────────────────────────────────┐
│ 如果在1000ms内没有喂狗           │
│ (没有调用LL_IWDG_Refresh)       │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│ 看门狗超时                      │
│ └─► 看门狗计时器溢出            │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│ 硬件系统复位                    │
│ └─► 处理器复位 (如同按下Reset)  │
└─────────────────────────────────┘
    │
    ▼
系统重新启动 (从Reset_Handler开始)

防护原理:
┌──────────────────────────────────────┐
│ main循环                             │
├──────────────────────────────────────┤
│ ... (IAP命令处理等)                 │
│ ... (可能某处死循环或错误)           │
│                                      │
│ sys_task()                           │
│ └─ iwdg_feed()                       │
│    ├─ 每次调用重置看门狗计数器       │
│    └─ 让看门狗再等待1000ms           │
└──────────────────────────────────────┘
      │
      └─► 正常循环: 每个循环周期调用
          一次iwdg_feed(), 看门狗永不
          超时,系统正常运行
```

## 状态机详解

### 1. IAP任务状态机

```
初始化完成 (sys_init -> iap_init)
    │
    ├─ iap_flag = false
    ├─ time_cnt = 0
    ├─ jump_flag = false
    │
    ▼
┌──────────────────────────┐
│   等待IAP命令状态        │
│   (iap_flag == false)    │
└──────────────────────────┘
    │
    ├─ [分支1] 接收到进入IAP命令 (cmd=1)
    │  └─► cmd_enter_iap()
    │      ├─ iap_flag = true
    │      └─ 进入"处理IAP命令状态"
    │
    └─ [分支2] 超时(20000次循环)
       └─► iap_task() 检测到超时
           ├─ iap_flash_verify()
           │  ├─ CRC32验证
           │  └─ 返回是否有效
           │
           ├─ [验证通过]
           │  └─► 进入"准备跳转状态"
           │
           └─ [验证失败]
              └─► 返回"等待IAP命令状态"
                  └─ time_cnt = 0 (重新计时)

┌──────────────────────────┐
│  处理IAP命令状态         │
│ (iap_flag == true)       │
└──────────────────────────┘
    │
    └─ 在此状态下:
       ├─ 等待上位机的IAP操作命令
       │  ├─ cmd=2: 读Flash
       │  ├─ cmd=3: 写Flash
       │  ├─ cmd=4: 写校验和
       │  └─ cmd=0: 退出IAP
       │
       └─ 当接收到cmd=0 (退出IAP)
          ├─ cmd_exit_iap()
          │  ├─ iap_flash_verify()
          │  │  └─ 验证新固件
          │  └─ jump_flag = true
          └─ 进入"准备跳转状态"

┌──────────────────────────┐
│  准备跳转状态            │
│ (jump_flag == true)      │
└──────────────────────────┘
    │
    └─ 在下一个iap_task()调用中
       ├─ 检测到jump_flag == true
       ├─ 禁用所有中断
       ├─ 反初始化定时器和看门狗
       ├─ jump_to_app()
       │  ├─ 读APP堆栈指针和入口地址
       │  ├─ 设置MSP
       │  └─ 跳转到APP (不返回)
       │
       └─ APP开始执行
          (BootLoader任务完成)

状态转换图:
    ┌─────────────────┐
    │ 等待IAP命令     │◄──────────┐
    │ (初始化)        │           │
    └────┬────────────┘           │
         │                        │
    ┌────▼────────┐        ┌──────┴─────┐
    │接收进入cmd? │──YES──►│处理IAP命令 │
    └────┬────────┘        └──────┬─────┘
         │ NO                     │
         │                   ┌────▼──────┐
    ┌────▼────────┐         │接收退出cmd?│
    │超时?        │──YES──►┌─► cmd=0    │
    └────┬────────┘        │ └───┬──────┘
         │ NO              │    │
         │                 │    ▼
         └─────────────────┼─►┌────────┐
                          │  │准备跳转 │
                          │  │ (验证) │
                          │  └───┬────┘
                          │      │
                          │      ▼
                          │   跳转APP
                          │  (不返回)
                          │
                          └─ 验证失败
                             返回等待
```

### 2. Modbus接收状态机

```
初始化: recv_state = 0

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STATE 0: 等待帧头
┌────────────────────────────┐
│  等待: 地址(1) + 功能码(1) │
│  计数: 0/2 字节            │
└────────────────────────────┘
    │
    ├─ [接收第1字节] 地址
    │   └─ cnt = 1
    │
    ├─ [接收第2字节] 功能码
    │   ├─ cnt = 2
    │   ├─ 检查功能码
    │   └─ recv_state = 1
    │
    └─ [超时] 如果1000ms无数据
       └─ recv_state = 0 (复位)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STATE 1: 根据功能码接收数据
┌────────────────────────────┐
│  功能码 = ?                │
└────────────────────────────┘
    │
    ├─ FNO_RD_REG (读寄存器)
    │   ├─ 接收8字节固定长度
    │   ├─ 数据 + CRC校验(2)
    │   └─ 完成后 recv_state = 2
    │
    ├─ FNO_WR_REG (写寄存器)
    │   ├─ 接收可变长度数据
    │   ├─ 先接收2字节长度
    │   ├─ 再接收[长度]字节数据
    │   ├─ 再接收2字节CRC
    │   └─ 完成后 recv_state = 2
    │
    ├─ FNO_IAP (IAP命令)
    │   ├─ 接收可变长度数据
    │   ├─ 先接收2字节长度
    │   ├─ 再接收[长度]字节数据
    │   ├─ 再接收2字节CRC
    │   └─ 完成后 recv_state = 2
    │
    └─ 其他功能码
        └─ 忽略/错误处理
           recv_state = 0 (复位)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STATE 2: 数据包完整
┌────────────────────────────┐
│  pkt_flag = true (标记)    │
│  等待主循环调用modbuss_task│
└────────────────────────────┘
    │
    ▼ (在modbuss_task()中)
┌────────────────────────────┐
│  pkt_decode()              │
│  ├─ CRC校验                │
│  ├─ 地址匹配检查           │
│  └─ 调用回调函数           │
└────────────────────────────┘
    │
    ▼
┌────────────────────────────┐
│  处理完成                  │
│  ├─ pkt_flag = false       │
│  ├─ recv_state = 0         │
│  └─ 返回等待状态           │
└────────────────────────────┘

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
