## 项目用途

### 主要功能

这是一个**嵌入式设备的在线固件升级系统**（BootLoader），主要功能包括：

#### 1. **固件在线更新 (IAP - In-Application Programming)**
- 支持无需硬件编程器的固件升级
- 通过通信接口（UART或CAN）接收新固件
- 将新固件编程到Flash存储器
- 验证新固件的完整性和正确性

#### 2. **多通信接口支持**
- **UART接口**: 115200bps串口通信
- **CAN接口**: 125Kbps CAN总线通信
- 可通过宏定义灵活切换

#### 3. **Modbus通信协议**
- 实现Modbus RTU从机功能
- 支持自定义IAP命令集
- 支持寄存器读写功能
- 标准CRC16校验

#### 4. **系统安全保护**
- 独立看门狗保护（1000ms超时）
- Flash CRC32完整性验证
- 通信数据CRC16校验
- 扇区级Flash擦除保护

#### 5. **应用程序管理**
- 管理两个Flash区域：BootLoader和应用程序
- 支持应用程序的启动跳转
- 支持应用程序的地址跳转

```

│        │   └─► 检查UART接收缓冲区是否有数据               │
│        ├─► modbuss_recv_byte(data)                         │
│        │   └─► 逐字节接收数据包，使用状态机处理           │
│        └─► modbuss_task()                                  │
│            ├─► 超时检查                                    │
│            └─► 数据包解码 (pkt_decode)                    │

│                                                             │

│        │           ├─► 禁用中断                            │
│        │           ├─► 反初始化外设                        │

│     3. sys_task()                                          │
```

### IAP更新流程

```
┌─────────────────────────────────────────────────────────────┐
│                    IAP固件更新流程                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│ 步骤1: 上位机检测设备                                       │
│   └─► 通过Modbus发送"进入IAP模式"命令                      │
│       └─► cmd=1, data=MCU地址                             │
│                                                             │
│ 步骤2: BootLoader进入IAP模式                               │
│   └─► 接收到cmd=1命令                                      │
│       ├─► cmd_enter_iap()                                 │
│       ├─► 设置iap_flag=true                               │
│       ├─► 停止自动跳转APP的超时计时                       │
│       └─► 返回ACK确认                                      │
│                                                             │
│ 步骤3: 上位机擦除旧固件                                     │
│   └─► 发送Flash擦除命令                                    │
│       └─► 实际上由写命令自动擦除相应扇区                  │
│                                                             │
│ 步骤4: 上位机分包发送新固件                                │
│   └─► 多次发送cmd=3 (写Flash)命令                         │
│       ├─► 地址: 0x08008000 (APP_BASE_ADDR)               │
│       ├─► 每包最多256字节                                 │
│       ├─► BootLoader处理: cmd_write_flash()              │
│       │   ├─► 如果是扇区起始地址，擦除扇区               │
│       │   ├─► 写入数据到Flash                             │
│       │   ├─► 验证写入数据                                │
│       │   └─► 返回写入结果                                │
│       └─► 重复直到整个固件传输完毕                        │
│                                                             │
│ 步骤5: 上位机写入校验信息                                   │
│   └─► 发送cmd=4 (写校验和)命令                            │
│       ├─► 写入应用程序大小                                │
│       ├─► 写入应用程序CRC32校验值                         │
│       ├─► 位置: ARG_BASE_ADDR (参数区)                    │
│       └─► BootLoader处理: cmd_write_checksum()           │
│                                                             │
│ 步骤6: 上位机要求BootLoader验证并启动APP                  │
│   └─► 发送cmd=0 (退出IAP模式)命令                         │
│       └─► BootLoader处理: cmd_exit_iap()                 │
│           ├─► 调用iap_flash_verify()                     │
│           │   ├─► 读取应用大小和CRC32                     │
│           │   ├─► 计算应用程序的CRC32                     │
│           │   ├─► 比较计算值与存储值                      │
│           │   └─► 返回验证结果                            │
│           ├─► [验证通过]                                   │
│           │   ├─► 设置jump_flag=true                      │
│           │   └─► 返回ACK                                │
│           └─► [验证失败]                                   │
│               └─► 返回失败应答 (cmd=0xFFFF)              │
│                   └─► 上位机重新发起升级流程              │
│                                                             │
│ 步骤7: BootLoader验证通过，下次循环跳转APP                │
│   └─► iap_task()检测到jump_flag=true                      │
│       ├─► 禁用所有中断                                    │
│       ├─► 反初始化定时器、看门狗等外设                    │
│       ├─► 调用jump_to_app()                              │
│       │   ├─► 从APP_BASE_ADDR读取堆栈指针                │
│       │   ├─► 从APP_BASE_ADDR+4读取程序入口地址          │
│       │   ├─► 设置MSP (主堆栈指针)                        │
│       │   └─► 跳转到程序入口 (不返回)                     │
│       └─► APP开始执行                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 关键参数

### 系统配置参数

| 参数 | 值 | 说明 |
|------|-----|------|
| **CPU频率** | 180MHz | ARM Cortex-M4内核频率 |
| **Flash大小** | 256KB (0x40000) | 总闪存容量 |
| **IRAM大小** | 128KB | 内部RAM总量 |
| **BootLoader大小** | 32KB (0x8000) | BootLoader占用空间 |
| **应用程序大小** | 224KB (0x38000) | 应用程序最大空间 |
| **参数区大小** | 4KB (0x1000) | 应用大小和CRC存储 |

### Flash地址分配

| 区间 | 大小 | 用途 |
|------|------|------|
| `0x08000000 - 0x08008000` | 32KB | BootLoader代码和数据 |
| `0x08007000 - 0x08008000` | 4KB | 参数区 (APP大小/CRC32) |
| `0x08008000 - 0x08040000` | 224KB | 应用程序 (APP) |

### 通信配置

**UART模式：**
| 参数 | 值 |
|------|-----|
| 串口号 | UART1 |
| 波特率 | 115200 bps |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验位 | 无 |
| 缓冲区 | 256字节 |

**CAN模式：**
| 参数 | 值 |
|------|-----|
| CAN接口 | CAN0 |
| CAN ID | 0xAA55 (29位扩展帧) |
| 波特率 | 125 Kbps |
| 数据长度 | 8字节/帧 |

### Modbus配置

| 参数 | 值 | 说明 |
|------|-----|------|
| 本地地址 | 2 | Modbus从机地址 |
| 远程地址 | 1 | PFC主机地址 |
| CAN本地ID | 0xAA55 | CAN标识符 |
| 通信超时 | 1000ms | 无数据超时清除 |
| 功能码 | 0x00 | 自定义IAP功能码 |

### 看门狗配置

| 参数 | 值 |
|------|-----|
| 预分频 | 32 |
| 超时 | 1000ms |
| 模式 | 复位模式 |

### 定时器配置

| 参数 | 值 |
|------|-----|
| 定时器 | TMR8 |
| 时钟源 | APB0时钟 |
| 周期 | 20微秒 |
| 频率 | 50KHz |
| 中断 | 启用 |

## 技术栈

### 硬件平台
- **处理器**: ARM Cortex-M4F (带FPU浮点单元)
- **芯片**: TAE32G5800 (泰利微TAE系列)
- **Flash**: 256KB 内部闪存
- **RAM**: 128KB 内部SRAM
- **外设**: UART, CAN, Timer, GPIO, IWDG, 等

### 开发工具
- **IDE**: Keil MDK-ARM (版本5.x+)
- **编译器**: ARM Compiler 6.14.1 (ARMCLANG)
- **汇编器**: ARM Assembler
- **链接器**: ARM Linker
- **调试器**: J-Link / ST-Link (通过Keil集成)

### 编程语言
- **主程序**: C语言
- **启动代码**: ARM汇编 (Thumb-2)
- **驱动库**: C语言

### 通信协议
- **Modbus RTU**: 主从通信协议
- **CRC校验**: CRC16 (通信), CRC32 (Flash)
- **物理层**: UART / CAN

### 编译配置
- **优化等级**: 可配置 (一般为-O2)
- **调试信息**: 启用 (-g3)
- **输出格式**: ELF, Hex, Bin
- **链接脚本**: `tae32g58xx_ac6_flash.sct`

### 依赖关系

```
编译依赖:
├── 设备包: Tai-Action.TAE32G58xx_DFP.1.3.1

├── C库: newlib
└── 链接脚本: AC6链接脚本格式

运行时依赖:
├── CMSIS-Core: v4.x (中断向量表, 系统初始化)

```
