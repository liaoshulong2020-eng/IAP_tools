```c
类型: 汇编/C混合函数

功能:
    处理硬件复位，系统启动的第一个程序

参数:
    无

返回值:

    硬件向量表(硬件复位时自动调用)

调用函数:

    - __PROGRAM_START() - C库入口点

工作流程:
    1. 初始化数据段 (从Flash复制初始化数据到RAM)
    2. 调用 SystemInit() 进行芯片级初始化
    3. 调用 __PROGRAM_START() 进入C程序
    4. 最终跳转到 main()

说明:
    这是复位后执行的第一个代码段，通常由编译器和链接器
    自动生成。这个函数不能被用户代码调用，只在硬件复位时
    由处理器自动调用一次。
```

### SystemInit()
```c
文件位置: Drivers/TAE32G58xx_Device/system_tae32g58xx.c:109
签名: void SystemInit(void)

功能:
    CMSIS系统初始化，执行芯片级配置

参数:
    无

返回值:
    void

调用者:
    Reset_Handler() - 启动代码

调用函数:
    无(直接操作寄存器)

主要操作:
    1. 配置中断向量表偏移
       └─ SCB->VTOR = (uint32_t)&__Vectors
    2. 禁用低电压检测 LVD
       └─ 防止在电源波动时误复位
    3. 启用FPU浮点单元
       └─ 启用CP10和CP11协处理器

关键代码:
    // 启用FPU
    #if (__FPU_PRESENT == 1) && (__FPU_USED == 1)
        SCB->CPACR |= ((3UL << 10*2) | (3UL << 11*2));
    #endif

调用时机:
    - 硬件复位后立即执行
    - 在 main() 之前执行
    - 只执行一次

说明:
    这个函数由CMSIS标准定义，用于执行与具体
    芯片相关的初始化。不同的芯片可能有不同的
    实现，但通常包含中断向量表配置和FPU启用。
```

## 主程序模块

### main()
```c
文件位置: main/main.c:23
签名: int main(void)

功能:
    系统主函数，执行初始化并进入主循环

参数:
    无

返回值:
    int (通常不返回)

调用者:
    __PROGRAM_START() / _start() - C库启动函数

调用函数:
    初始化阶段:
    - SystemClock_Config() - 配置系统时钟
    - LL_Init() - LL库初始化
    - delayms(20) - 延时等待
    - sys_init() - 系统模块初始化

    主循环中调用:
    - modbus_iap_task() - Modbus IAP任务
    - iap_task() - IAP处理任务
    - sys_task() - 系统维护任务
    - uart1_tx_poll() (UART模式) - 发送轮询
    - cans_rx_poll() (CAN模式) - 接收轮询

执行流程:
    ```
    main()
    {
        // 1. 时钟配置
        SystemClock_Config();

        // 2. LL库初始化
        LL_Init();

        // 3. 延时稳定
        delayms(20);

        // 4. 系统初始化
        sys_init();

        // 5. 主循环
        while(1)
        {
            // IAP通信任务
            modbus_iap_task();

            // IAP处理和跳转
            iap_task();

            // 系统任务(喂狗)
            sys_task();

            // 通信接口轮询
            #if USE_UART
                uart1_tx_poll();
            #elif USE_CAN
                cans_rx_poll();
            #endif
        }
    }
    ```

说明:
    main() 是C程序的标准入口点。这个函数执行
    系统初始化后进入一个无限循环，在循环中不断
    执行三个主要任务。这个函数通常不返回。
```

### SystemClock_Config()
```c
文件位置: main/main.c:50
签名: static void SystemClock_Config(void)

功能:
    配置系统时钟频率为180MHz

参数:
    无

返回值:
    void

调用者:
    main() - 主函数

调用函数:
    - LL_RCU_SysclkInit() - 配置系统时钟
    - SystemCoreClockUpdate() - 更新时钟变量
    - LL_RCU_Pll1Cfg() - 配置PLL1
    - LL_RCU_HRPWM_ClkCfg() - HRPWM时钟配置
    - LL_RCU_ADC_ClkCfg() - ADC时钟配置
    - LL_RCU_CAN_ClkCfg() - CAN时钟配置

时钟树:
    外部晶振 (12MHz)
    └─► PLL0 (x15) ──► SYSCLK (180MHz)

    PLL1 (x30) ──► 360MHz
    ├─► HRPWM (÷2) ──► 180MHz
    ├─► ADC (÷6) ──► 60MHz
    └─► CAN (÷3) ──► 120MHz

时钟配置结果:
    - 系统SYSCLK: 180MHz (PLL0 x15)
    - PLL1: 360MHz (用于外设)
    - HRPWM: 180MHz (PLL1/2)
    - ADC: 60MHz (PLL1/6)
    - CAN: 120MHz (PLL1/3)

说明:
    这个函数配置系统的核心时钟频率。系统采用
    外部12MHz晶振，通过PLL倍频到180MHz。
    不同外设使用不同的时钟分频，以适应不同的
    工作频率要求。
```

### iap_task()
```c
文件位置: main/iap.c:229
签名: void iap_task(void)

功能:
    IAP主任务，在主循环中调用

参数:
    无

返回值:
    void

调用者:
    main() 主循环 - 每个主循环周期调用一次

调用函数:
    - request_enter_iap() (UART模式) - 请求进入IAP
    - iap_flash_verify() - 验证Flash
    - LL_IWDG_DeInit() - 反初始化看门狗
    - LL_TMR_DeInit() - 反初始化定时器
    - jump_to_app() - 跳转到应用程序

执行逻辑:
    ```
    iap_task()
    {
        // 1. 检查是否已进入IAP模式
        if (iap_flag)
            return;  // 已进入IAP模式，等待命令

        // 2. UART模式: 周期性发送进入IAP请求
        #if USE_UART
            request_enter_iap();
        #endif

        // 3. 超时检查(约400ms)
        if (time_cnt++ >= 20000)
        {
            // 4. 验证Flash应用程序
            if (iap_flash_verify())
            {
                // 5. 验证通过，准备跳转
                LL_IWDG_DeInit();   // 关看门狗
                LL_TMR_DeInit();    // 关定时器
                jump_to_app();      // 跳转APP(不返回)
            }
            else
            {
                // 验证失败，重新开始
                time_cnt = 0;
            }
        }
    }
    ```

说明:
    这是IAP任务的主函数，负责：
    1. UART模式下主动请求进入IAP
    2. 超时自动跳转到应用程序
    3. 验证应用程序的完整性
```

### iap_flash_verify()
```c
文件位置: main/iap.c:197
签名: bool iap_flash_verify(void)

功能:
    验证Flash中应用程序的完整性

参数:
    无

返回值:
    bool
    - true: 应用程序有效，可以跳转
    - false: 应用程序无效或不存在

调用者:
    - iap_task() - IAP任务
    - cmd_exit_iap() - 退出IAP命令

调用函数:
    - crc32_update() - 计算CRC32校验值

验证过程:
    1. 从ARG_BASE_ADDR (参数区)读取应用大小
    2. 从ARG_BASE_ADDR+4读取存储的CRC32值
    3. 检查应用大小是否在合法范围内(< APP_MAX_SIZE)
    4. 计算应用程序区(APP_BASE_ADDR)的CRC32
    5. 比较计算值与存储值是否相等
    6. 返回比较结果

数据布局:
    ARG_BASE_ADDR+0: [应用大小, 4字节]
    ARG_BASE_ADDR+4: [应用CRC32, 4字节]

说明:
    这个函数实现了Flash应用程序的完整性检验。
    它通过对比存储的CRC32值来确认应用程序
    是否有效。这是IAP系统的安全保障。
```

```c
跳转步骤:
    1. 禁用所有中断
    2. 从APP_BASE_ADDR (0x08008000)读取:
       - offset+0: MSP (主堆栈指针初始值)
       - offset+4: PC (程序计数器/程序入口地址)
    3. 调用__set_MSP()设置堆栈指针
    4. 通过函数指针跳转到应用程序入口
    5. 应用程序开始执行(此函数不返回)

代码示例:
    typedef void (*pFunction)(void);

    uint32_t AppAddr = APP_BASE_ADDR;
    uint32_t AppStack = *(uint32_t*)AppAddr;
    pFunction AppEntry = (pFunction)(*(uint32_t*)(AppAddr + 4));

    __disable_irq();
    __set_MSP(AppStack);
    AppEntry();  // 跳转到应用程序

说明:
    这个函数实现了从BootLoader到应用程序
    的跳转。ARM Cortex-M处理器在启动时会
    自动从固定地址读取堆栈指针和复位向量。
```

```c
    - LL_EFLASH_Verify() - 验证数据

编程流程:
    ```
    flash_program(addr, buff, size)
    {
        1. 计算扇区索引
           sec_idx = (addr - FLASH_BASE) / SECTOR_SIZE

        2. 检查是否需要擦除
           if (addr % SECTOR_SIZE == 0)
               LL_EFLASH_EraseSector(sec_idx)  // 擦除整个扇区

        5. 返回成功
           return true;
    }
    ```

    - 大小必须8字节的倍数
    - 不能跨扇区边界

    自动在扇区边界处进行擦除，然后编程数据，
```

```c
        ├─ 设置pkt_flag = true

        └─ 不重置状态(等待解码完成)

    逐字节接收数据，当接收完整个数据包时
    设置pkt_flag标志，等待解码处理。
```
